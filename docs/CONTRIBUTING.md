# Contributing for ZorHUB

Thanks for your interest in contributing to ZorHUB.

ZorHUB is currently in early development. Contributions should follow the project goals:
- Friendly user experience.
- Safe system actions.
- Clear explanations.
- No arbitrary command execution.
- Respect for Linux/Zorin OS conventions.
- Modular architecture.

## Contribution types

You cam contribute with:
- Bug reports. (IR/Issue report)
- Feature ideas. (PR/Pull Request)
- Documentation improvements.
- UI/UX suggestions.
- Linux compatibility testing.
- Extentions.

## Before proposing a feature

A feature should answer at least one of these questions:
- Does it make Linux easier to understand?
- Does it unify a fragmented system task/program?
- Does it improve safety?
- Does it make ZorHUB more visuals or accessible?
- Does it preserve advanced technical details when needed?

## Code style

Code should be readable, modular, and documented.

Avoid:
- Large single-file implementations.
- Arbitrary shell command execution.
- Unsafe privileged actions.
- UI elements without clear purpose.

## Disclamer

PR and IR will open as the project is developed.
