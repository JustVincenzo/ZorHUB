# Versioning

ZorHUB uses a  staged versioning model.

## Private pre-alpha

```text
pα0.x.PATCH
```

Examples:

```text
pα0.1
pα0.2.8
pα0.3.4
```

Private pre-alpha builds are not public releases. THey are incomplete, unstable and may contain simuled behavior.

## Alpha

Format:
```text
αYY.M.x
```

Example:

```text
α26.8.1
α26.10.25
α26.2.9
```

Alpha boulds may be public, but most include a strong instability warning.

## Beta

Format:

```text
βYY.M.x
```

Beta builds are public testing builds with a more stable feature set.

## Gamma/stable releases

Format:

```text
γMAJOR.MINOR.PATCH
```

Example:

```text
γ1.0.0
γ4.2.10
γ2.4.1
```

Gamma releases are stable public releases intended for general use.
